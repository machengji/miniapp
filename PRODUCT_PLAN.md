# Aletheia 产品规划与核心功能演进

本文档旨在记录 "Aletheia - 深度荣格心理分析师" 小程序的后续功能迭代方向、商业化策略及技术路线图。

## 🌟 核心功能规划 (Core Features)

为了构建“记录 -> 分析 -> 沉淀 -> 疗愈”的完整闭环，计划新增以下四大核心模块：

### 1. 梦境日记 (Dream Journal) - 数据沉淀
**目标**：将用户的一次性对话转化为持久化的个人心理档案。
*   **功能详情**：
    *   **时间轴列表**：按日期展示历史梦境。
    *   **结构化输入**：用户输入梦境的同时，选择情绪标签（恐惧、焦虑、平静等）和清晰度。
    *   **AI 自动摘要**：分析完成后，自动生成 10 字以内的标题（如“高塔与坠落之梦”）。
*   **技术支撑**：微信云开发数据库 (Collections: `dreams`)。

### 2. 长期记忆与关联分析 (Long-term Memory) - 智能增强
**目标**：模拟真实心理分析师的“长期记忆”，识别跨时间的心理模式。
*   **功能详情**：
    *   **记忆检索**：在分析新梦境时，自动检索用户过去 3-5 次的梦境关键词。
    *   **对比洞察**：AI 指出梦境间的联系（例如：“这周你第二次梦见‘蛇’，但这次它是温顺的，暗示焦虑在缓解”）。
*   **技术支撑**：向量数据库或关键词匹配，动态 System Prompt 注入。

### 3. 心灵映射图 (Soul Map) - 可视化反馈
**目标**：将抽象的心理状态具象化，增加产品的可读性和分享率。
*   **功能详情**：
    *   **原型雷达图**：基于荣格 12 原型（天真者、孤儿、照顾者...）的当前状态分布。
    *   **Persona/Shadow 占比**：展示社会面具与阴影自我的冲突程度。
*   **技术支撑**：ECharts 或 Canvas 绘图。

### 4. 每日共时性卡片 (Synchronicity Card) - 促活机制
**目标**：利用“共时性”概念提升用户每日打开率。
*   **功能详情**：
    *   每日抽取一张指引卡片。
    *   **正面**：荣格风格的抽象意象画（AI 生成或预设图库）。
    *   **背面**：基于用户近期梦境分析生成的个性化“洞察语”。

## 💰 商业化路径 (Commercialization)

### 1. 流量变现 (入门)
*   **激励视频广告**：
    *   **逻辑**：用户可免费查看基础回复，点击“查看深度潜意识推理 (Chain of Thought)”需观看 15-30 秒广告。
    *   **优势**：利用用户对“AI 怎么想的”好奇心，点击率高。

### 2. 增值服务 (进阶)
*   **“洞察币” (Token 计费)**：按次付费进行深度模型分析。
*   **VIP 订阅**：
    *   解锁无限次对话。
    *   解锁“长期记忆”关联分析功能。
    *   解锁更多分析师人格（如弗洛伊德视角）。

### 3. 内容付费
*   **月度心理报告**：生成精美的 PDF 报告。
*   **冥想/解梦课程**：音频付费内容。

## 📅 开发路线图 (Roadmap)

### Phase 1: 基础建设 (当前阶段)
*   [ ] **数据库搭建**：设计 `dreams` 和 `users` 表结构。
*   [ ] **保存功能**：实现聊天记录到云数据库的存储逻辑。
*   [ ] **历史记录页**：开发简单的时间轴页面用于查看过往记录。

### Phase 2: 智能与记忆
*   [ ] **上下文增强**：修改云函数，支持读取历史记录并在 Prompt 中携带上下文。
*   [ ] **标签系统**：自动提取梦境关键词并存储。

### Phase 3: 体验与视觉
*   [ ] **数据可视化**：引入图表库展示“心灵映射图”。
*   [ ] **共时性卡片**：开发每日抽卡功能。
*   [ ] **UI 升级**：优化打字机效果与深色模式适配。
